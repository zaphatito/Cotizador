SISTEMA DE COTIZACIONES — REGISTRO DE CAMBIOS

Versión: 2.0
Fecha: 2026-02-24

Resumen
- Nueva versión enfocada en envío al API, control de estados, experiencia visual y mayor seguridad en el flujo de cotización.

Cambios y mejoras
- [Mejora] Se agregó el envío de cotizaciones al API desde el historial, con adjuntos PDF y CMD.
- [Mejora] Si el código de una cotización no es válido para el API, ahora se regenera automáticamente el PDF/CMD antes de reenviar.
- [Mejora] Se agregó el nuevo estado "Reenviado" y su gestión completa en historial.
- [Mejora] Ahora puedes personalizar los colores de estado (Pagado, Por pagar, Pendiente, Reenviado y No aplica).
- [Mejora] Se incorporó tema de interfaz configurable: Sistema, Claro u Oscuro.
- [Mejora] Se renovó la interfaz principal e historial para mejorar legibilidad y navegación.
- [Mejora] Se guarda el tamaño y posición de las ventanas (historial y cotizador) entre sesiones.
- [Mejora] Se actualizó la lógica de precios a P. Máx / P. Min / P. Oferta, incluyendo selector visual de precio.
- [Mejora] Mejoró la selección de presentaciones para elegir la variante correcta según contexto.
- [Mejora] En Perú, el ticket ahora muestra totales de botellas/esencias y el total general de la cotización.
- [Mejora] En PDF se mejoró el bloque de observaciones para evitar cruces con el área de totales.
- [Mejora] Se añadieron validadores y ejemplos de formato para documento y teléfono por país.
- [Mejora] Se optimizó el autocompletado/búsqueda para respuestas más rápidas.

Correcciones
- [Fix] Se corrigió la reapertura de cotizaciones históricas para respetar correctamente el tipo de precio guardado.
- [Fix] Se evitó el reenvío duplicado al API sin confirmación explícita del usuario.
- [Fix] Se reforzó la validación del catálogo para evitar crear/abrir cotizaciones con datos incompletos o inválidos.
- [Fix] Se ajustaron migraciones de base de datos para mayor compatibilidad en actualizaciones y bases nuevas.

Notas importantes
- Si aparece "Catálogo inválido", usa "Actualizar productos" antes de crear o abrir cotizaciones.
- Después de un envío exitoso al API, la cotización puede cambiar automáticamente al estado "Reenviado".
- El tema visual y los colores de estado quedan guardados para próximas sesiones.

Compatibilidad
- No se requieren acciones adicionales para continuar usando el sistema.
- La base de datos local y la configuración del usuario se conservan.

Soporte
- Si notas algún comportamiento inesperado después de actualizar, reinicia la aplicación y vuelve a intentar.
- Si el problema persiste, llama al soporte técnico.
